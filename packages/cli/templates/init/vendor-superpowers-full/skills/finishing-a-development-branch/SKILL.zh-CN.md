# 开发分支收尾

## 目标

在实现完成后，统一完成：

- 最终验证
- 结果汇报
- 分支处理决策

## 流程

1. 运行最终验证
2. 汇总当前工作状态
3. 向 human partner 提供分支处理选项
4. 根据选择执行后续动作

## 典型选项

- 合并到主分支
- 保留分支，稍后继续
- 创建 PR
- 放弃这条分支 / worktree

## 必须确认

- 测试或验证已经重新运行
- 当前 diff 与目标一致
- 没有混入无关改动
- human partner 明确知道接下来会发生什么

## 不要做的事

- 没验证就宣称工作完成
- 不经过确认就自行合并
- 脏 diff 状态下直接收尾
- 不说明选项就擅自删除 worktree

## 输出要求

至少说明：

- 做了什么
- 验证结果如何
- 当前分支 / worktree 状态
- 可以选择的后续动作
