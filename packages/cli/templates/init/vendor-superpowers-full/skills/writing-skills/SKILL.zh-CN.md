# 编写 Skills

## 目标

skill 不是普通文档，而是会塑造 agent 行为的“行为代码”。  
本 skill 用来指导如何新增或修改 skill，并通过测试证明它真的提升了结果。

## 核心理念

- skill 应有清晰触发条件
- skill 应明确告诉 agent 何时使用、如何使用、何时停止
- 行为 shaping 的文案要慎改
- skill 修改必须经过评估，而不是靠感觉

## 编写原则

1. 先定义这个 skill 解决什么失败模式
2. 明确触发条件
3. 明确不可违反的硬规则
4. 提供流程、清单和红旗
5. 补上测试 / 压测方法

## 编写要求

- 触发条件写清楚
- 不要写成泛泛建议
- 重点内容要用强约束语言
- 给出红旗和常见借口，防止 agent 自我合理化
- 有必要时提供模板和示例

## 测试 skill

skill 写完后不能只靠阅读判断好坏。  
要在真实或模拟任务中压测，看它是否：

- 真能触发
- 真能改变行为
- 没带来新副作用

## 修改现有 skill 时

- 先理解原设计哲学
- 不要轻易改动关键措辞
- 需要有 before/after 的证据

## 底线

把 skill 当成行为程序来设计，而不是当成说明文来润色。
